Que a Web é feita de conteúdo, nós já sabemos. O que fazemos atualmente é enviar conteúdos de uma
máquina para outra, mas máquinas não dão significados e sentidos às coisas,inclusive as
linguagens, somente após ao passar pela percepção humana, as máquinas conseguem reproduzir esse
significado das coisas.O que se vem estudando nesses 20 anos de web semântica, é como padronizá-la,
para que o sentido de todas as linguagens proporcionem uma melhor experiência para o usuário.
Como exemplo, podemos colocar como comparativo, a voz do Google Maps e a Alexia (fica pra outro texto),
tanto uma ferramenta quanto outra, utilizam de conteúdo, o óbvio, mas o sentido que dão aos signos e as
linguagens, criam o que Saussure define como linguagem: ( SAUSSURE Ferdinand)” A linguagem é
social
e individual; psíquica; psico-fisiológica e física. Portanto, a fusão de Língua e Fala. Para ele, a
Língua é definida como a parte social da linguagem e que só um indivíduo não é capaz de mudá-la”.

Sob essa perspectiva é que são criados os nossos smartphones, redes sociais, entre outras tecnologias.
A linguagem é a mesma e cada área de conhecimento, vai contribuindo à sua maneira.

As palavras tem que ir além da sua intenção gramatical quando fazemos um código em html, utilizando
as tags usamos a semântica, e cada tag tem um sentindo, um significado, que será na Web
uma linguagem visual. Metalinguagem, para Metadados. A estrutura é feita pela função das tags
“head”, “title”, “p”, “ font”, “font color”, e a linguagem de programação JS, por exemplo, permite
implementar itens de alto nível em páginas da web, incluindo animações, fotos, imagens,
gráficos, mapas ou informações que se atualizam em intervalos de tempo.

Como organizar esses códigos a partir da Semântica, tornando-os padrão, para que faça sentindo para as
máquinas?

Como trabalhar a padronização das tags dentro do contexto de Web Semântica? Na sua primeira versão o
HTML era uma página estática, a interação do usuário com a informação era receptiva-passiva.

A Web semântica tem como principal objetivo, desenvolver tecnologias e linguagens que tornem a
informação legível para as máquinas. Mas a informação com sentido.

Ao citar a frase:”Eu bebo leite de cabra em pó”, quis exemplificar como funciona o significado para a
máquina. Temos também a semiótica dos textos que vai tratar a linguagem. Eu entendo que temos que
recorrer também à semiótica Santaella, Lucia. “Semiótica é o estudo dos signos, que consistem em todos os
elementos que representam algum significado e sentido para o ser humano, abrangendo as linguagens
verbais e não-verbais.” fazendo analogia a Língua, o UX (design, writing, research), seriam a literatura,
onde temos movimentos literários, estilos, contextos históricos e sociais, usando o atributo da Língua como
um produto dinâmico, social e individual, respeitando as regras de morfologia, sintaxe, ao
transformar esses conceitos em tags, poderia ser um pontapé para criar sentido dos dados processados pelas
máquinas.

Sim e semiótica também, como não falar do sentido que o indivíduo atribui a um signo? E essa atribuição
passa por sua experiência? Qual a sua experiência? Apesar de ser muito usada no campo de design ela vai dar
visual para o seu conteúdo semântico então entendemos que ambas trabalham juntas para uma melhor experiência do usuário na web proporcionando
autonomia e uma utilização mais abrangente da tecnologia e contribuindo para a Inteligência Artificial.

“A semiótica é a ciência que investiga todas as linguagens possíveis, ou seja, tem como objetivo o
exame dos modos de constituição de todo e qualquer fenômeno como fenômeno de produção de significação
e de sentido” (SANTAELLA, 1985, p. 15).

Temos a semântica que vai tratar do significado dos signos, esse significado ele pode ser definido pela
morfologia (análise morfológica) ou pela posição desse vocábulo em uma sintaxe (análise sintática). A
grande preocupação hoje em relação a web semântica é a padronização. Já temos definido
na própria ciência do significado o que cada linguagem, cada signo , querem proporcionar,
como utilizá-los. Tem vastos estudos. Podemos relacionar os signos e suas funções, com as tags e sintaxe de
HTML. O trabalho não é individual, tratemos a Web Semântica, assim como Saussure, definiu a
Linguagem, social, individual e dinâmica.

Como trabalhar a padronização das tags dentro do contexto de Web Semântica?

Ainda não sabemos, e o trabalho é conjunto. Quando usávamos o HTML para HotSite, a comunicação
era unilateral. Quando este artigo, exemplifica com a frase:”Eu bebo leite de cabra em pó”, foi reproduzido o
recorte de uma fala sem o contexto, e as máquinas não conseguem fazer esse recorte, que vai tratar a linguagem,
os usuários que devem inserir um contexto, para que haja inteligência artificial.
Então um texto de uma linguagem não-formal, foi re-significado para uma linguagem formal, respeitando as regras de
morfologia, sintaxe, para que o significado tivesse sentido. Esses complementos gramaticais, além de dados, são
aspectos inerentes à própria língua.

A implementação de recursos semânticos ainda não obedece a uma estrutura definida. Mas começar a pensar
semântica e semiótica dentro de uma linguagem FrontEnd, é uma proposta.

Você sabe o que é Semântica? Você sabe qual o significado dessa palavra e sua importância para o mundo da
tecnologia? Ouviu falar sobre Web Semântica? Quando você faz o desenvolvimento de sites, como aplica
os seus códigos? Qual o sentido?

Conceituando Semântica como a ciência do significado, a ciência que dá significado ao signo.

Na frase: ”Eu bebo leite de cabra em pó”, ou “Eu bebo o leite em pó de cabra” (BARROS-Diana-Luz-Pessoa-
de.-Teoria- Semiótica-do-Texto). Na primeira frase “Eu bebo leite em pó de cabra”, o que foi dito?Qual o
significado do que está descrito? E se utilizarmos a imagem para representar um indivíduo tirando o leite da cabra,
processando para que fique em pó, e por último ingerido? Fez sentido? Mudando a frase para “Eu
tiro o leite da cabra, faço um processamento para que fique em pó, e para bebê-lo, acrescento água,
temos um significado sem ruídos.
